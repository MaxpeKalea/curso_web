Sections

Esto trata de elementos de seccionado, que son los que se reflejan en el esquema/outline de la página.

Inicialmente estos elementos incluían <body> y los encabezados, que van de <h1> a <h6>.

En HTML 5 se añaden los nuevos elementos semánticos:

Section, Header, Footer, Nav, Article, Aside.

Section

El elemento de HTML section (<section>) representa una sección genérica de un documento. Sirve para determinar qué contenido corresponde a qué parte de un esquema. Piensa en el esquema como en el índice de contenido de un libro; un tema común y subsecciones relacionadas. Es, por lo tanto, una etiqueta semántica.
Su funcionalidad principal es estructurar semánticamente un documento a la hora de ser representado por parte de un agente usuario. Por ejemplo, un agente de usuario que represente el documento en voz, podría exponer al usuario el índice de contenido por niveles para navegar rápidamente por las distintas partes.

Header

El elemento de HTML Header (<header>) representa un grupo de ayudas introductorias o de navegación. Puede contener algunos elementos de encabezado, pero también otros elementos como un logo, una sección que aglutine secciones de encabezados, un formulario de búsqueda o cosas parecidas.

Footer

El Elemento HTML Footer (<footer>) representa un pie de página para el contenido de sección más cercano o el elemento raíz de sección (p.e, su ancestro mas cercano article, aside, nav, section, blockquote, body, details, fieldset, figure, td).
Un pie de página típicamente contiene información acerca de el autor de la sección, datos de derechos de autor o enlaces a documentos relacionados.

Nav

La MDN indica que esta etiqueta crea un menú de navegación (https://developer.mozilla.org).
El elemento HTML Nav (<nav>) representa una sección de una página cuyo propósito es proporcionar enlaces de navegación, ya sea dentro del documento actual o a otros documentos. Ejemplos comunes de secciones de navegación son menús, tablas de contenido e índices.

Article

El Elemento article de HTML (<article>) representa una composición auto-contenida en un documento, página, una aplicación o en el sitio, que se destina a distribuir de forma independiente o reutilizable, por ejemplo, en la sindicación.
Podría ser un mensaje en un foro, un artículo de una revista o un periódico, una entrada de blog, un comentario de un usuario, un widget interactivo o gadget, o cualquier otro elemento independiente del contenido.

Aside

El Elemento HTML Aside (<aside>) representa una sección de una página que consiste en contenido que está tangencialmente relacionado con el contenido que le rodea, que podría ser considerado independiente de ese contenido. Estas secciones son a menudo representadas como barras laterales o como inserciones y contienen una explicación al margen como una definición de glosario, elementos relacionados indirectamente, como publicidad, la biografía del autor, o en aplicaciones web, la información de perfil o enlaces a blogs relacionados.
https://developer.mozilla.org/es/docs/Web/HTML/Elemento

Grouping content

Son etiquetas que por defecto tienen un display bloque, pero no contribuyen al esquema de la página.

Párrafos

Se pueden crear párrafos con las etiquetas <p>, <pre> y <blockquote>.

Ejemplo de un párrafo preformateado (<pre>):

                                Esto esta dentro
                                de la etiqueta pre

Se puede usar para fragmentos de código:

                                let x = 25
                                let y = 24
                                let z = x + y

La etiqueta blockquote se utiliza para crear citas literales en un bloque, tal y como hicimos al principio del curso.

La utilizamos para respetar los derechos de autor de la página original

Listas

Existen 3 tipos de lista:

  • Desordenada: <ul>
  • Ordenada: <ol>
  • De definiciones: <dl>

Una lista ordenada es la que por naturaleza tiene un orden (los días de la semana).

Ejemplo de lista ordenada:

  1. Lunes
  2. Martes
  3. Miércoles
  4. Jueves
  5. Viernes
  6. Sábado
  7. Domingo

Una lista desordenada es la que puedo organizar como quiera.

Ejemplo de lista desordenada:

  • Desordenada: <ul>
  • Ordenada: <ol>
  • De definiciones: <dl>

Bloques neutros

La etiqueta <div> no tiene ningún valor semántico.

Unida al uso de las clases CSS es el principal elemento de diseño en la Web.

Bloques nuevos en HTML

  • <Address>
  • <Main>
  • <Figure>

Según aparece en la Web de MDN: El elemento HTML <address> aporta información de contacto para su <article> mas cercano o ancestro <body>; en el último caso lo aplica a todo el documento.

Normalmente puede ser colocado dentro de un elemento <footer> de la sección

El elemento HTML <Main> representa el contenido principal del <body> de un documento o aplicación. El área principal del contenido consiste en el contenido que está directamente relacionado, o se expande sobre el tema central de un documento o la funcionalidad central de una aplicación. Este contenido debe ser único al documento, excluyendo cualquier contenido que se repita a través de un conjunto de documentos como barras laterales, enlaces de navegación, información de derechos de autor, logos del sitio y formularios de búsqueda (a menos, claro, que la función principal del documento sea un formulario de búsqueda).

El elemento <Figure> es un contenedor de imágenes del que hablaremos al referirnos a éstas.

Otras etiquetas relacionadas con bloques

La etiqueta <hr> representa un cambio de tema entre párrafos, creando una línea horizontal.

La etiqueta <br> crea un salto de línea.

Estas etiquetas es recomendable usarlas lo menos posible, por su bajo valor semántico y porque pueden estropear el diseño.

Elementos en línea neutros

La etiqueta <span> no tiene ningún valor semántico. Es el equivalente en línea a <div>, que era en bloque.

Por ejemplo si quiero que la palabra verde salga de su color.

Elementos claramente semánticos

  • <Strong>

    • Palabra clave, mejora SEO.

  • <Em>

    • Poner énfasis, mejora accesibilidad.

  • <Sup>

    • Superíndice, tiene que ser de uso generalizado y reconocible.

  • <Sub>

    • Subíndice, también tiene que ser de uso generalizado y reconocible.

  • <Small>

    • Lo que se dice "la letra pequeña" del texto. Introduce información legal o similar.

Elementos con valor semántico nuevo

  • Bold o <b>

    • Destacado con fines utilitarios.

  • Italics o <i>

    • Calidad diferente de texto, incluyendo cambio de tono o de estado de ánimo, un termino técnico, una designación taxonómica, un refrán, un pensamiento, el nombre de un barco, una frase idiomática en otro lenguaje... Una cursiva sin énfasis, alternativa a <Em>.

  • Underline o <u>

    • Texto escrito de manera incorrecta voluntariamente, nombres propios chinos.

  • Strike o <s>

    • Tachar algo que ya no es correcto, como un precio rebajado o retractarse de una frase.

Elementos de tipo técnico

  • <Samp>

    • Sample, aparece como cursiva.

  • <Code>

    • Code, aparece como cursiva.

  • <Kbd>

    • Keyboard, aparece como espaciada.

  • <Var>

    • Variables, aparece como cursiva.

  • <Dfn>

    • Definition

  • <Abbr>

    • Abbr

    Nos quedamos con la etiqueta <abbr>, que indica abreviaturas o acrónimos, y nos da la opción de especificar el significado de éstos. Esto mejora SEO ya que si alguien busca las palabras sin abreviar aparecen las nuestras, y accesibilidad, ya que un lector de voz puede explicarlas.

Uso de referencias y citas

  • <Q> (quote)

    • Es lo mismo que un <Blockquote> pero en línea.

  • <Cite> (o referencia)

    • Títulos de libros, obras, creaciones en general a las que se haga referencia, incluyendo las páginas Web.

Otras etiquetas semánticas de HTML 5

  • <Time>

    • Por ejemplo, para indicar que la frase "el día de la Constitución de este año" es una fecha.

  • <Mark>

    • Marcado interactivo, no se debe usar a través de VSC, ya que quien la suele generar es el propio programa backend.

Edits

  • <Ins>

    • Instead, muestra el contenido que se ha insertado en lugar de otro.

  • <Del>

    • Deteled, muestra el contenido que se ha borrado en su lugar, o simplemente borrado.
